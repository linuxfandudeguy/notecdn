NoteCDN API Documentation

Welcome to the NoteCDN API documentation. NoteCDN provides API endpoints for fetching package versions, package files, and entry points.

API Endpoints

NoteCDN API endpoints:

  • /api/v1/packages/npm/{package} - Get versions of a package
  • /api/v1/packages/npm/{package}@{version} - Get package files for a specific version
  • /api/v1/packages/npm/{package}@{version}/entrypoints - Get entry points for a package version

Usage Example

Example usage of the NoteCDN API:

https://uncors.vercel.app/?url=https://api-notecdn.vercel.app/api/v1/packages/npm/axios

This fetches the versions of the axios package through the CORS proxy.

https://uncors.vercel.app/?url=https://api-notecdn.vercel.app/api/v1/packages/npm/axios@0.21.1

This fetches the package files for version 0.21.1 of axios through the CORS proxy.

https://uncors.vercel.app/?url=https://api-notecdn.vercel.app/api/v1/packages/npm/axios@0.21.1/entrypoints

This fetches the entry points for version 0.21.1 of axios through the CORS proxy.

Integration in Your Application

To integrate NoteCDN API in your application, construct the API URL with the appropriate endpoint and parameters, prefixed by the CORS proxy URL.

Example:

// JavaScript example using fetch API with CORS proxy
fetch('https://uncors.vercel.app/?url=https://api-notecdn.vercel.app/api/v1/packages/npm/axios@0.21.1')
    .then(response => response.json())
    .then(data => {
        console.log(data);
    })
    .catch(error => {
        console.error('Error fetching data:', error);
    });
